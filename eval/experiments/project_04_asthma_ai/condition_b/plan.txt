### 3. DATABASE SCHEMA

**Users Table**
```sql
CREATE TABLE users (
  id INTEGER PRIMARY KEY,
  email TEXT UNIQUE NOT NULL,
  google_id TEXT UNIQUE,
  name TEXT,
  location TEXT,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  last_login TIMESTAMP,
  settings_json TEXT  -- {theme, notifications_enabled, etc}
);

CREATE INDEX idx_users_email ON users(email);
```

**Logs Table**
```sql
CREATE TABLE logs (
  id INTEGER PRIMARY KEY,
  user_id INTEGER NOT NULL FOREIGN KEY REFERENCES users(id),
  severity INTEGER CHECK (severity BETWEEN 1 AND 10),
  symptoms TEXT,  -- JSON array: ["cough", "shortness_of_breath", ...]
  notes TEXT,
  latitude REAL,
  longitude REAL,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_logs_user_created ON logs(user_id, created_at);
```

**Predictions Table**
```sql
CREATE TABLE predictions (
  id INTEGER PRIMARY KEY,
  user_id INTEGER NOT NULL FOREIGN KEY REFERENCES users(id),
  risk_score REAL CHECK (risk_score BETWEEN 0.0 AND 1.0),
  risk_level TEXT CHECK (risk_level IN ('low', 'moderate', 'high')),
  factors_json TEXT,  -- {weather: {...}, history: {...}, air_quality: {...}}
  advice TEXT,
  recommended_actions TEXT,  -- JSON array
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  calendar_event_id TEXT  -- Google Calendar event ID
);

CREATE INDEX idx_predictions_user_created ON predictions(user_id, created_at DESC);
```

**Google Tokens Table**
```sql
CREATE TABLE google_tokens (
  id INTEGER PRIMARY KEY,
  user_id INTEGER NOT NULL UNIQUE FOREIGN KEY REFERENCES users(id),
  access_token TEXT,
  refresh_token TEXT,
  expires_at TIMESTAMP,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

**Model Metadata Table**
```sql
CREATE TABLE model_metadata (
  id INTEGER PRIMARY KEY,
  model_version TEXT,
  training_date TIMESTAMP,
  num_samples INTEGER,
  accuracy REAL,
  feature_names TEXT,  -- JSON array
  model_path TEXT,  -- e.g., "models/model_v1.pkl"
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

### 4. API ENDPOINTS

**Authentication**
- `POST /auth/google` — Google OAuth callback; exchange code for token
- `POST /auth/refresh` — Refresh Google access token
- `POST /auth/logout` — Clear session

**Logs**
- `POST /logs` — Submit new severity + symptoms (auth required)
- `GET /logs` — Fetch user's log history (auth required)
- `GET /logs/{log_id}` — Get single log details

**Predictions**
- `GET /predictions/latest` — Get latest prediction for user (auth required)
- `GET /predictions` — Get prediction history (auth required)

**Calendar Sync**
- `POST /calendar/sync` — Manually trigger sync to Google Calendar (auth required)
- `GET /calendar/status` — Check sync status

**Admin** (internal only)
- `POST /admin/retrain` — Trigger model retraining
- `GET /admin/health` — System health check (DB, Redis, scheduler)

### 5. ML PIPELINE

**Model Architecture:**
- **Estimator:** RandomForestClassifier (n_estimators=100, max_depth=10)
- **Target:** Binary classification (flare-up risk: 0=low, 1=high)
  - Definition: flare-up = average severity over next 7 days ≥ 6
- **Features (30–40 total):**
  - **Symptom history (14 days):** mean/max/min/count of severity, symptom frequency
  - **Weather (current + forecast):** temperature, humidity, pressure, pollen (if available)
  - **Air quality (current):** AQI, PM2.5, PM10, ozone
  - **Time features:** day-of-week, season, time-since-last-log
  - **User demographics:** age_group (derived from registration), location
  - **Interaction terms:** severity × AQI, humidity × symptoms

**Feature Extraction:**
- All features normalized (StandardScaler)
- Missing values imputed with median (for historical) or 0 (for current)
- Categorical features one-hot encoded

**Training Data:**
- **Mock generation:** 200 synthetic logs per test user (600 total)
  - Realistic patterns: higher severity on high-AQI days, seasonal peaks
  - Varied symptoms: cough, shortness_of_breath, wheezing, chest_tightness, nighttime_symptoms
- **Initial training:** Mock data only (train/val split 80/20)
- **Periodic retraining:** Weekly merge of mock + accumulated real user logs

**Model Persistence:**
- Save trained model to `models/model_{date}.pkl` after each training
- Load latest model at startup; fall back to mock model if missing

### 7. DEPLOYMENT CHECKLIST

**Local Development (no Docker):**
1. Clone repo; create venv
2. `pip install -r backend/requirements.txt`
3. `cd frontend && npm install`
4. Create `.env` with Google OAuth credentials, OpenWeatherMap API key (optional)
5. Run `./startup.sh`:
   - Initialize SQLite DB (create tables)
   - Generate mock data
   - Train initial ML model
   - Start Redis in background (`redis-server`)
   - Start Celery worker (`celery -A tasks.celery_app worker --loglevel=info`)
   - Start FastAPI server (`uvicorn app.main:app --host 0.0.0.0 --port 8000`)
   - Start React dev server (`cd frontend && npm start`)

**Unattended Operation (Linux systemd):**
- Create systemd services:
  - `asthma-api.service` (FastAPI + gunicorn)
  - `asthma-celery-worker.service` (Celery worker)
  - `asthma-celery-beat.service` (Celery beat scheduler)
  - `asthma-redis.service` (Redis)
- All services configured to auto-restart on crash (Restart=always)
- Watchdog timeout: 30s (kill and restart if hung)
- Logging: JSON to syslog or local file

**Error Recovery:**
- Celery DLQ: Failed tasks logged but don't block others
- API failures: Graceful degradation (use cached data, return 503 if critical)
- Token expiry: Re-auth prompt on frontend; log to error queue

### 9. KEY DECISIONS SUMMARY

| Decision | Rationale | Risk Mitigation |
|----------|-----------|-----------------|
| SQLite + SQLAlchemy | Zero-ops, cloud-ready ORM | No backup (acceptable for prototype) |
| FastAPI + async | Fast, modern, handles concurrent requests | Structured exception handling |
| Celery + Redis | Decouples long-running work; local and scalable | DLQ for failed tasks; watchdog restart |
| Classical ML (RandomForest) | Interpretable, fast training, works with small data | Weekly retraining on real data; mock data representative |
| Free APIs + caching | Reduces cost; cache handles rate limits | Exponential backoff + graceful degradation |
| Google OAuth | Single sign-on; official Calendar API | Token refresh retry; force re-auth on failure |
| React + Chakra UI | Warm, accessible UX; TypeScript prevents bugs | Component testing; accessibility audits optional |
